# API Referansı

- Kaynak gerçek olarak kodu, şemayı ve çalışan davranışı kullan; tahminden alan üretme.
- Öğeleri kararlı ad ve hiyerarşiyle sırala; kısa amaç cümlesinden sonra imza/söz dizimi ver.
- Parametre türü, zorunluluk, varsayılan, sınır, dönüş, hata ve yan etkiyi açıkla.
- Her öğeye minimal çalışan örnek ekle; örneğin çıktısını ve ön koşulunu göster.
- Ortak kavramı tek yerde belgeleyip bağla; referansta uzun öğretici anlatı kullanma.
- Eksik/deprecated öğeleri ve sürüm farkını işaretle, örnekleri test veya doğrulama ile senkron tut.
