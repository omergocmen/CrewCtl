# Git Commit Yazımı

- Tek bir mantıksal değişikliği seç; ilgisiz dosyaları veya kullanıcıya ait değişiklikleri dahil etme.
- Proje convention'ını kontrol et; yoksa kısa, emir kipinde ve sonucu anlatan bir başlık yaz.
- Conventional Commits kullanılıyorsa doğru tür ve yalnızca yararlı scope seç.
- Gövdede ne değiştiğini tekrarlamak yerine nedenini, davranış etkisini ve önemli ödünleşimi açıkla.
- Kırıcı değişiklik ve issue bağlantısını projenin beklediği biçimde belirt.
- Commit öncesi staged farkı yeniden oku ve ilgili doğrulamayı çalıştır; test edilmemiş iddiada bulunma.
