# Pull Request Yazımı

- Başlığı kullanıcı etkisini anlatan tek bir değişikliğe odakla.
- Özette sorunu, çözüm yaklaşımını ve neden bu kapsamın seçildiğini kısa yaz.
- Davranış değişikliklerini, önemli dosyaları ve özellikle incelenmesi gereken riskli noktaları belirt.
- Çalıştırılan testleri gerçek komut ve sonuçla listele; çalıştırılmayan kontrolü açıkça söyle.
- UI değişiminde görsel kanıtı, veri/API değişiminde migration ve uyumluluk notunu ekle.
- İlgili issue'yu bağla; büyük log, dosya listesi veya commit mesajlarını aynen tekrarlama.
