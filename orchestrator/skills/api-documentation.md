# API Dokümantasyonu

- Okuyucunun ilk başarılı isteğini en kısa yoldan çalıştıracağı başlangıç örneği ver.
- Base URL, kimlik doğrulama, gerekli header ve ortam farklarını açıkla; gerçek sır kullanma.
- Her endpoint için amaç, parametre, istek, yanıt, durum kodu, hata ve idempotency davranışını göster.
- Örnekleri geçerli, kopyalanabilir ve şemayla tutarlı tut; sahte alan uydurma.
- Sayfalama, rate limit, retry, webhook ve sürümleme gibi çapraz davranışları tek yerde belgeleyip bağla.
- Dokümanı çalışan sözleşme/testle karşılaştır ve kod değişikliğiyle birlikte güncelle.
